# Roadmap

_Last updated: 2026-03-22 (America/Chicago)_

This roadmap describes the public direction of World Runtime.

## Current Priorities

1. Stabilize the public runtime core around replay, simulation, policy, connectors, and observability.
2. Keep the Public API `/v1`, SDK starter, and App Server protocol coherent and well-documented.
3. Improve extension seams so new adapters and connector plugins can be added without kernel rewrites.
4. Promote only those adapters, examples, and playbooks that are ready for public maintenance.

## Near-Term Public Work

- tighten public docs and onboarding
- expand adapter-extension guidance
- harden compatibility and release gates
- improve connector and persistence operational playbooks

## Longer-Term Direction

- add more public-ready adapters over time
- deepen operator and governance workflows on the stable runtime surfaces
- keep private incubator work separate until it is ready for promotion
