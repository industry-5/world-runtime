# Security Policy

## Reporting a Vulnerability

Please report suspected vulnerabilities privately by emailing **security@world-runtime.dev**.

Include:
- Affected version/commit
- Reproduction steps or proof of concept
- Impact assessment
- Any suggested mitigation

Do not open public GitHub issues for active security vulnerabilities.

## Response Targets

- Initial acknowledgement: within 2 business days
- Triage decision: within 5 business days
- Status updates: at least weekly until resolution

## Supported Versions

Security updates are provided for:
- Latest release candidate series (`1.0.0-rc.x`)
- Latest stable major version after GA

Older versions may be declined or asked to upgrade first.
