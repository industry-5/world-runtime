# digital-twin-mini

- Public adapter: `adapter-digital-twin`.
- Premise: attach a host-bound digital-twin overlay to the `power_grid` contingency path first, then extend the same advisory layer into `city_ops`.
- Overlay bundle: this directory carries the overlay runtime artifact set plus `host_bindings.json` to keep host attachment explicit.
- Supplemental proof: `overlay_options.json` preserves unsafe and power-grid-first alternatives beside the approved cross-host overlay path.
