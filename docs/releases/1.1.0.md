# World Runtime 1.1.0 Release Notes

Release date: 2026-04-23 (America/Chicago)

## Summary

World Runtime 1.1.0 promotes the completed v1.1 enhancement train into the public release line. The release expands the supported story beyond the original v1.0 core by making the runtime easier to consume as a pinned dependency, easier to supervise locally, and easier to inspect through supported management surfaces.

## Highlights

- Supported package-hosted consumption through `python -m world_runtime serve` and `world-runtime serve`
- Managed runtime host and service supervision with validated service manifests and lifecycle diagnostics
- Provider inventory and task-profile routing with deterministic selection and bounded fallback
- Supported runtime-admin inventory, provider inspection, task resolution, and bounded reconcile through the App Server, Public API `/v1`, and starter SDK
- Local AI structured-extraction reference stack validated through the released runtime surfaces

## Compatibility

- App Server protocol remains major-compatible within `1.x`
- Public API remains additive within `/v1`
- Python SDK starter continues to track the supported `/v1` surface
- Persistence migrations remain forward-only and immutable once released
- Extension contracts remain governed by the documented compatibility matrix and validation checks
